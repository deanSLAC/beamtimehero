Next Steps — PostgreSQL and Deployment Prep for bllogs_converter
================================================================

1. DATABASE TABLES
   Tables are auto-created by ensure_tables() on first run.
   No manual DDL needed — just ensure the DB user has CREATE TABLE permission.
   If running in the existing bl152 database with the bl152 user, this should
   already be the case (same setup as bldata_converter).

2. API_KEY ENVIRONMENT VARIABLE
   LLM error detection requires the API_KEY env var (Stanford AI Gateway).
   Add this to the pod's environment via Flux/k8s secrets if not already present.
   Without it, LLM detection is skipped and regex fallback is used automatically.

3. VACUUM / MAINTENANCE
   The log_commands table will grow over time (one row per SPEC command).
   Estimate: ~500-2000 commands per day during active beamtime.
   - Enable autovacuum (on by default in PostgreSQL)
   - Monitor table size periodically:
       SELECT pg_size_pretty(pg_total_relation_size('"BL15-2_log_commands"'));
   - Consider VACUUM ANALYZE monthly if query performance degrades

4. RETENTION POLICY
   After 6-12 months, old log command data may no longer be useful.
   Options:
   - DELETE FROM "BL15-2_log_commands" WHERE timestamp < NOW() - interval '12 months';
   - Or archive to a separate table/export before deleting
   - Same for log_errors table

5. TABLE PARTITIONING (if needed)
   If log_commands exceeds ~1M rows, consider partitioning by month:
       CREATE TABLE "BL15-2_log_commands" (...) PARTITION BY RANGE (timestamp);
   This would require recreating the table — plan for a maintenance window.
   Not needed initially.

6. SCHEDULING
   The script should be called every 60 seconds. This is handled externally
   (k8s CronJob, systemd timer, or sleep loop in a pod).
   The script itself is stateless between runs — all state is in PostgreSQL.

7. DEPENDENCIES
   The module uses the same dependencies as the main app:
   - psycopg2-binary (already in requirements.txt)
   - requests (for LLM API calls — add to requirements.txt if not present)
   No new pip packages needed beyond requests.

8. TESTING
   Test with --dry-run and --no-llm first:
       python -m bllogs_converter --dry-run --no-llm
   Then without --dry-run to verify DB writes:
       python -m bllogs_converter --no-llm
   Then with LLM (requires API_KEY):
       python -m bllogs_converter
